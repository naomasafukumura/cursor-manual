# 5. ブランチを作成する

「ブランチ」とは、メインのファイルに影響を与えずに、自分だけの作業スペースを作る機能です。
**必ずブランチを作ってから編集を始めましょう。**

## なぜブランチが必要？

- **メインのファイル（mainブランチ）を壊さない** ためです
- 自分の変更が完成したら、レビューを受けてからメインに反映します
- チームメンバーが同時に違う作業をしても衝突しません

## ブランチを作成する

### 方法1：Cursorの画面から作成（おすすめ）

1. **Cursorの左下**を見ると、現在のブランチ名（`main`）が表示されています

   ::: info UIの場所
   ブランチ名はCursorウィンドウの左下のステータスバーに表示されています。分岐アイコンの横に「main」と書かれています。
   :::

2. **ブランチ名をクリック**
   上部にブランチ選択メニューが表示されます

3. **「+ Create new branch...」をクリック**

4. **ブランチ名を入力** して Enter を押します

### 方法2：ターミナルから作成

```bash
git checkout -b ブランチ名
```

例：
```bash
git checkout -b feature/update-lesson-5
```

## ブランチの命名規則

チームで統一するために、以下のルールでブランチ名をつけてください：

| やること | ブランチ名の例 |
|---------|--------------|
| 新しい機能を追加 | `feature/追加する内容` |
| 既存の内容を修正 | `fix/修正する内容` |
| ドキュメントの更新 | `docs/更新する内容` |

**例：**
- `feature/add-lesson-10` （レッスン10を追加）
- `fix/typo-in-lesson-5` （レッスン5の誤字を修正）
- `docs/update-readme` （READMEを更新）

::: warning 注意
- ブランチ名は **英語・小文字** で書いてください
- スペースの代わりに **ハイフン `-`** を使ってください
- 日本語は使わないでください
:::

## 現在のブランチを確認する

Cursor の左下に表示されているブランチ名が、**`main` 以外**になっていればOKです。
