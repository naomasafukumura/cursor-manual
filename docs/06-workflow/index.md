# 6. 毎日の作業方法

ここからが毎回の作業です。**全部 Claude Code に日本語で話しかけるだけ** です。

## ① Cursorでプロジェクトを開く

1. Cursorを起動
2. **「Open project」** をクリック → プロジェクトフォルダを選んで **「開く」**
3. 「Trust the authors?」と出たら → **「Yes, I trust the authors」**
4. **Terminal → New Terminal** でターミナルを表示

::: tip すでにプロジェクトを開いている場合
Cursorに前回のプロジェクトが残っていればそのままでOKです。ターミナルだけ開いてください。
:::

## ② プロジェクトフォルダにいることを確認

```bash
cd ~/Documents/プロジェクト名
```

例：
```bash
cd ~/Documents/kpi-dashboard
```

::: tip 毎回入力するのが面倒な場合
ターミナルを開いたら **上矢印キー（↑）** を押すと、前に打ったコマンドが出てきます。
:::

## ③ Claude Code を起動する

```bash
claude --dangerously-skip-permissions
```

::: info この長いコマンドは何？
`claude` だけでも起動できますが、`--dangerously-skip-permissions` をつけると、毎回「許可しますか？」と聞かれなくなります。
操作がスムーズになるのでおすすめです。
:::

「Welcome back!」と表示されたら起動成功です！

## ④ まず最新にする

```
「更新ある？あったら最新にして」
```

他のメンバーが変更していたら、自動で最新版を取ってきてくれます。
**作業を始める前に必ずやってください。**

## ⑤ ファイルを編集する

```
「DASHBOARD.html のタイトルを変えて」
「このファイルの誤字を見つけて直して」
「新しいファイルを追加して」
```

日本語で具体的に指示するだけ。Claude が編集してくれます。

## ⑥ 変更を送る

```
「コミットしてプッシュして」
```

これでGitHubにアップロードされます。
Vercelに連携しているリポジトリなら、サイトも自動で更新されます。

::: danger プッシュは絶対に忘れないで！
プッシュを忘れると、他の人が「更新ある？」と聞いてもあなたの変更が見えません。
結果、同じ場所を同時に編集してしまい、ファイルがぶつかります（コンフリクト）。
**作業が終わったら必ず「コミットしてプッシュして」を言ってください。**
:::

## まとめ：覚えるのはこの3つだけ

| やること | Claude に言うこと |
|---------|-----------------|
| 最新にする | 「更新ある？」 |
| 編集する | 「ここを直して」 |
| 送る | 「コミットしてプッシュして」 |

## Claude Code を終わるとき

- `/exit` と打つ
- または **`Ctrl + C`** を2回押す

→ 困ったときは [トラブルシューティング](/troubleshooting) を見てください
→ わからない言葉は [用語集](/glossary) を見てください
