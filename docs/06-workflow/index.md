# 6. 毎日の作業方法

ここからが毎回の作業です。**全部 Claude Code に日本語で話しかけるだけ** です。

## Claude Code を起動する

1. Cursorでプロジェクトを開く（スタート画面の「Recent projects」からクリック）
2. **`Ctrl + `` `** でターミナルを開く
3. 以下を打って Enter：
   ```bash
   claude
   ```

## 毎回の流れ

### ① まず最新にする

```
「更新ある？あったら最新にして」
```

他のメンバーが変更していたら、自動で最新版を取ってきてくれます。
**作業を始める前に必ずやってください。**

### ② ファイルを編集する

```
「DASHBOARD.html のタイトルを変えて」
「このファイルの誤字を見つけて直して」
「新しいファイルを追加して」
```

日本語で具体的に指示するだけ。Claude が編集してくれます。

::: warning 確認してから承認
Claude が「変更していいですか？」と聞いてきたら、**内容を見てから** 「y」（はい）を押してください。
よくわからなければ「n」（いいえ）でキャンセルできます。
:::

### ③ 変更を送る

```
「コミットしてプッシュして」
```

これでGitHubにアップロードされます。
Vercelに連携しているリポジトリなら、サイトも自動で更新されます。

### ④ 大きな変更のときは「ブランチ」を使う

AIでデザインを大幅に変えるなど、大きな変更をするときは：

```
「ブランチ作って、ダッシュボードのデザインを変えて」
```

::: info ブランチって何？
**下書きノート** のようなものです。
- 本番ファイル（main）= **教室の黒板**
- ブランチ = **自分の下書きノート**

下書きノートで作業するので、黒板（本番）は壊れません。
完成したら確認してもらってから黒板に書き写します。
:::

ブランチで作業が終わったら：

```
「プルリクエスト作って」
```

これで他のメンバーに「確認してください」と依頼できます。
確認がOKなら、本番に反映されます。

## まとめ：覚えるのはこの5つだけ

| やること | Claude に言うこと |
|---------|-----------------|
| 最新にする | 「更新ある？」 |
| 編集する | 「ここを直して」 |
| 送る | 「コミットしてプッシュして」 |
| 大きな変更 | 「ブランチ作って、〇〇して」 |
| 確認依頼 | 「プルリクエスト作って」 |

## Claude Code を終わるとき

- `/exit` と打つ
- または **`Ctrl + C`** を2回押す

→ わからない言葉は [用語集](/glossary) を見てください
