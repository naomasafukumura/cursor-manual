# 6. 毎日の作業方法

ここからが毎回の作業です。**全部 Claude Code に日本語で話しかけるだけ** です。

## ① Cursorでプロジェクトを開く

Cursorを起動すると、スタート画面が表示されます。

![Cursorスタート画面](/images/cursor-start.svg)

- **初めて開く場合** → **「Open project」** をクリック → 編集したいフォルダを選んで **「開く」**
- **2回目以降** → **「Recent projects」** に前回のプロジェクトが表示されるので、**クリックするだけ**

::: info Open project / Clone repo / Connect via SSH の3つのボタン
スタート画面の上に3つのボタンがあります。
- **Open project** = 自分のパソコンにあるフォルダを開く
- **Clone repo** = GitHubからダウンロードする（ステップ5で説明済み）
- Connect via SSH = 使いません
:::

## ② ターミナルを開く

プロジェクトが開いたら、画面上のメニューから **「Terminal」→「New Terminal」** をクリック。

画面の下に黒い画面（ターミナル）が表示されます。

## ③ Claude Code を起動する

ターミナルに以下をコピー＆ペーストして **Enter**：

```bash
claude --dangerously-skip-permissions
```

::: info この長いコマンドは何？
`claude` だけでも起動できますが、`--dangerously-skip-permissions` をつけると、毎回「許可しますか？」と聞かれなくなります。
操作がスムーズになるのでおすすめです。
:::

「Welcome back!」と表示されたら起動成功です！

![Claude Code 起動画面](/images/claude-running.svg)

## ④ まず最新にする

```
「更新ある？あったら最新にして」
```

他のメンバーが変更していたら、自動で最新版を取ってきてくれます。
**作業を始める前に必ずやってください。**

## ⑤ ファイルを編集する

```
「DASHBOARD.html のタイトルを変えて」
「このファイルの誤字を見つけて直して」
「新しいファイルを追加して」
```

日本語で具体的に指示するだけ。Claude が編集してくれます。

## ⑥ 変更を送る

```
「コミットしてプッシュして」
```

これでGitHubにアップロードされます。
Vercelに連携しているリポジトリなら、サイトも自動で更新されます。

::: danger プッシュは絶対に忘れないで！
プッシュを忘れると、他の人が「更新ある？」と聞いてもあなたの変更が見えません。
結果、同じ場所を同時に編集してしまい、ファイルがぶつかります（コンフリクト）。
**作業が終わったら必ず「コミットしてプッシュして」を言ってください。**
:::

## まとめ：覚えるのはこの3つだけ

| やること | Claude に言うこと |
|---------|-----------------|
| 最新にする | 「更新ある？」 |
| 編集する | 「ここを直して」 |
| 送る | 「コミットしてプッシュして」 |

## Claude Code を終わるとき

- `/exit` と打つ
- または **`Ctrl + C`** を2回押す

→ わからない言葉は [用語集](/glossary) を見てください
