# 7. 変更を送る

編集が終わったら、変更した内容をチームに送りましょう。
3つのステップがあります。

| ステップ | やること | イメージ |
|---------|---------|---------|
| ① 選ぶ | 送りたいファイルを選ぶ | 封筒にファイルを入れる |
| ② 名前をつける | 何を変えたか一言メモする | 封筒にメモを貼る |
| ③ 送る | GitHubに送信する | 封筒をポストに入れる |

## ① 送りたいファイルを選ぶ

1. Cursorの左側にある **分岐したマーク**（上から3番目のアイコン）をクリック

   ::: info どこにあるの？
   左端のアイコンが縦に並んでいるバーの、上から3番目です。線が枝分かれしたようなマークです。
   :::

2. 「Changes」の下に、編集したファイルが一覧で出ています
3. 送りたいファイルの横の **「+」ボタン** をクリック

::: tip 全部送りたいとき
「Changes」の横にある **「+」** をクリックすると、全ファイルまとめて選べます。
:::

## ② 何を変えたか書く

1. 上にあるテキスト入力欄に、**何を変えたか一言で書く**

   **良い例：**
   - `レッスン5の内容を更新`
   - `新しいファイルを追加`
   - `誤字を修正`

   **悪い例：**
   - `更新` ← 何を更新したかわからない
   - `あああ` ← 意味がわからない

## ③ 送る

1. **「✓ Commit」ボタン** をクリック
2. そのあと **「...」メニュー** → **「Push」** をクリック

::: info 「ブランチを公開しますか？」と出たら
**「OK」** を押してください。初めて送るときだけ出るメッセージです。
:::

## 送れたか確認

「Changes」の下が **空っぽ** になっていればOK！

## うまくいかないとき

**「rejected」と出た場合：**
他の人が先に変更を送っていたときに出ます。ターミナルに以下を打って Enter → もう一度プッシュ：
```bash
git pull --rebase
```
