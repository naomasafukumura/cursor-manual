# 7. コミット & プッシュ

編集・保存したファイルを、GitHubにアップロードする手順です。
これを行うことで、チームメンバー全員が最新のファイルを見れるようになります。

## 全体の流れ

```
編集・保存 → ステージング → コミット → プッシュ
```

| 用語 | 意味 |
|------|------|
| **ステージング** | 「この変更をGitHubに送ります」と選ぶこと |
| **コミット** | 変更内容に名前をつけて記録すること |
| **プッシュ** | 記録した変更をGitHubにアップロードすること |

## ステップ1：ソース管理タブを開く

1. Cursor の左側にある **ソース管理アイコン**（分岐のマーク）をクリック

   ::: tip スクリーンショット
   📸 ここにソース管理アイコンのスクリーンショットが入ります
   :::

2. 「変更」セクションに、編集したファイルが一覧で表示されます

## ステップ2：変更をステージングする

1. ステージングしたいファイルの横にある **「+」ボタン** をクリック

   ::: tip スクリーンショット
   📸 ここにステージングボタンのスクリーンショットが入ります
   :::

2. ファイルが「ステージされている変更」セクションに移動します

::: info すべてのファイルをまとめてステージング
「変更」セクションの横にある **「+」ボタン** をクリックすると、すべてのファイルを一度にステージングできます
:::

## ステップ3：コミットする

1. 上部の **メッセージ入力欄** に、変更内容を簡潔に書きます

   **コミットメッセージの書き方：**
   - ✅ `レッスン5の内容を更新`
   - ✅ `新しい教材ファイルを追加`
   - ✅ `誤字を修正`
   - ❌ `更新`（具体的でない）
   - ❌ `あああ`（意味がない）

2. **「✓ コミット」ボタン** をクリック（または `Cmd + Enter` / `Ctrl + Enter`）

   ::: tip スクリーンショット
   📸 ここにコミットボタンのスクリーンショットが入ります
   :::

## ステップ4：プッシュする

コミットした後、変更をGitHubにアップロードします。

1. ソース管理タブの **「...」メニュー** をクリック
2. **「Push」** をクリック

   ::: tip スクリーンショット
   📸 ここにプッシュメニューのスクリーンショットが入ります
   :::

::: info 初回プッシュ時
「ブランチを公開しますか？」と聞かれたら **「OK」** をクリックしてください。
これは、新しいブランチをGitHubに初めてアップロードするときに表示されます。
:::

## プッシュが成功したか確認する

- ソース管理タブの変更一覧が **空** になっていればOK
- GitHubのリポジトリページをブラウザで開いて、自分のブランチが表示されていれば成功です

## よくあるエラー

### 「rejected」と表示される場合

他のメンバーが先に変更をプッシュしている場合に起きます。

**解決方法：**
```bash
git pull --rebase
```
をターミナルで実行してから、もう一度プッシュしてください。

### 認証エラーの場合

Personal Access Token が期限切れの可能性があります。
[Gitの初期設定](/03-git-setup/#personal-access-token-の作り方) を参考にトークンを再作成してください。
