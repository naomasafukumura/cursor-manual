# 8. プルリクエストを作成する

プッシュした変更を、メインのファイル（mainブランチ）に反映するためには、
**プルリクエスト（PR）** を作成して、レビューを受ける必要があります。

## プルリクエストとは？

- 「この変更をメインに反映していいですか？」とチームに確認するための仕組みです
- レビューを受けることで、ミスを防ぎ、品質を保てます

## プルリクエストを作成する

### 手順

1. **GitHubにアクセス**
   ブラウザで Masa-English のリポジトリページを開きます

2. **「Compare & pull request」ボタンが表示されている場合**
   プッシュした直後であれば、黄色いバナーが表示されているので、
   **「Compare & pull request」** をクリックします

   ::: tip スクリーンショット
   📸 ここにCompare & pull requestバナーのスクリーンショットが入ります
   :::

3. **バナーが表示されていない場合**
   - **「Pull requests」** タブをクリック
   - **「New pull request」** をクリック
   - **「compare:」** のドロップダウンから自分のブランチを選択
   - **「Create pull request」** をクリック

4. **タイトルと説明を入力**

   | 項目 | 書く内容 | 例 |
   |------|---------|-----|
   | **Title（タイトル）** | 変更内容を簡潔に | `レッスン5の内容を更新` |
   | **Description（説明）** | 変更の詳細 | `レッスン5に新しい例文を3つ追加しました` |

   ::: tip スクリーンショット
   📸 ここにPR作成フォームのスクリーンショットが入ります
   :::

5. **レビュアーを設定**
   右側の **「Reviewers」** で、レビューしてほしい人を選択します

6. **「Create pull request」をクリック**

## プルリクエストがマージされるまで

1. レビュアーがあなたの変更を確認します
2. コメントや修正依頼が来ることがあります
3. 修正が必要な場合は、同じブランチで修正 → コミット → プッシュ するだけでOK（PRは自動で更新されます）
4. レビューが承認されたら、**マージ**（メインに反映）されます

::: info マージ後のブランチ
マージが完了したら、使ったブランチは自動的に削除されることがあります。
次の作業では、新しいブランチを作成してください。
:::

## プルリクエスト後に mainブランチを最新にする

マージが完了したら、自分のローカル環境も最新にしましょう。

### 手順

1. **mainブランチに切り替え**
   Cursor の左下のブランチ名をクリック → **「main」** を選択

2. **最新の変更を取得**
   ターミナルで以下を実行：
   ```bash
   git pull
   ```

3. これで自分のパソコンも最新の状態になります
