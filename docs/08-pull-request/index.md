# 8. 確認してもらう

変更を送ったら、他のメンバーに **「これで大丈夫か見てください」** とお願いします。
これを「**プルリクエスト（PR）**」と呼びます。

## プルリクエストを作る

1. ブラウザで **GitHubのリポジトリページ** を開く
2. 黄色いバーが出ていたら **「Compare & pull request」** をクリック

   ::: info 黄色いバーが出ない場合
   「**Pull requests**」タブ → **「New pull request」** → 自分のブランチを選んで **「Create pull request」**
   :::

3. **タイトル** に何を変えたか書く（例：`レッスン5の内容を更新`）
4. **説明欄** に詳しい内容を書く（例：`新しい例文を3つ追加しました`）
5. 右側の **「Reviewers」** で確認してほしい人を選ぶ
6. **「Create pull request」** をクリック

これで完了！あとはレビュアーの返事を待ちましょう。

## コメントや修正依頼が来たら

1. 指摘された部分を **Cursorで修正**
2. また [7. 変更を送る](/07-commit-push/) と同じ手順で送る
3. 自動的にプルリクエストが更新されます（新しく作り直す必要はありません）

## 承認されたら

レビュアーが「OK」を出すと、あなたの変更が本番ファイルに反映されます。

そのあと、自分のパソコンも最新にしておきましょう：

1. Cursorの左下のブランチ名をクリック → **「main」** を選ぶ
2. ターミナルで以下を打って Enter：
   ```bash
   git pull
   ```

これで最新の状態になります！
次の作業をするときは、また [5. 作業スペースを作る](/05-branch/) からスタートしてください。
