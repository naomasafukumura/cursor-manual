# 用語集

Cursor、Git、GitHubに関する用語を、わかりやすく解説します。

## エディタ関連

### Cursor（カーソル）
AIが搭載されたコードエディタ。ファイルの編集やGitの操作ができるツールです。
Visual Studio Code（VS Code）をベースに作られています。

### ターミナル
コマンド（文字の命令）を入力して、パソコンを操作するための画面。
Cursorの中に内蔵されていて、`Ctrl + `` ` で開けます。

### エクスプローラー
Cursorの左側に表示される、ファイルやフォルダの一覧画面のことです。

## Git 関連

### Git（ギット）
ファイルの変更履歴を管理するためのツール。
「いつ・誰が・何を変えたか」を記録してくれます。

### リポジトリ（Repository）
ファイルと変更履歴をまとめて保管する場所。
「プロジェクトのフォルダ」のようなものです。

### クローン（Clone）
GitHubにあるリポジトリを、自分のパソコンにコピー（ダウンロード）すること。

### ブランチ（Branch）
メインのファイルに影響を与えずに、自分だけの作業スペースを作る機能。
木の「枝」のようなイメージです。

### main ブランチ
チームの「公式」バージョンが入っているブランチ。
ここに直接変更を加えず、別のブランチで作業してからマージします。

### ステージング（Staging）
「この変更をコミットに含めます」と Git に伝えること。
変更したファイルを「+」ボタンでステージングします。

### コミット（Commit）
変更内容に名前（メッセージ）をつけて、ローカルに記録すること。
ゲームの「セーブ」に似ています。

### プッシュ（Push）
ローカル（自分のパソコン）のコミットを、GitHubにアップロードすること。

### プル（Pull）
GitHubにある最新の変更を、自分のパソコンにダウンロードすること。

### マージ（Merge）
ブランチの変更を、別のブランチ（通常はmain）に統合すること。

### コンフリクト（Conflict）
2人以上が同じファイルの同じ部分を編集した場合に起きる「衝突」。
手動で解決する必要があります。困ったら管理者に相談してください。

## GitHub 関連

### GitHub（ギットハブ）
Gitリポジトリをインターネット上で管理・共有するためのサービス。
チームでファイルを共同編集するための「オンラインの共有フォルダ」のようなものです。

### プルリクエスト（Pull Request / PR）
「この変更をメインに反映していいですか？」とチームに確認するための仕組み。
レビューを受けてからマージされます。

### レビュー（Review）
プルリクエストの内容を他のメンバーが確認すること。
コメントや修正依頼をもらえます。

### Organization（オーガニゼーション）
GitHubの「組織」機能。チームのリポジトリを一元管理できます。
Masa English の場合は `Masa-English` という Organization があります。

### Personal Access Token
GitHubにアクセスするための「パスワードの代わり」となるトークン（文字列）。
パスワードよりもセキュリティが高い方法です。

## AI 関連

### Claude Code（クロード・コード）
ターミナルで使えるAIアシスタント。
自然な日本語で指示を出すと、ファイルの編集を手伝ってくれます。

## その他

### Vercel（バーセル）
ウェブサイトを公開するためのサービス。
GitHubと連携して、変更を自動的に反映してくれます。

### Markdown（マークダウン）
テキストに簡単な記号をつけて、見出しやリスト、太字などを表現する書き方。
`.md` という拡張子のファイルで使います。

**よく使うMarkdown記法：**

| 書き方 | 表示 |
|--------|------|
| `# 見出し1` | 大きい見出し |
| `## 見出し2` | 中くらいの見出し |
| `**太字**` | **太字** |
| `- リスト` | リスト項目 |
| `[リンク](URL)` | リンク |
