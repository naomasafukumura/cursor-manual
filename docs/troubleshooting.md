# トラブルシューティング

よくあるトラブルと対処法をまとめています。

## 緊急ボタン

操作中に「おかしいな？」と思ったら、以下の2つを覚えておいてください。

| 状況 | やること | ボタン |
|------|---------|--------|
| Claude が長時間動いていて終わらない | 今の処理を中断する | **`Esc`** キーを押す |
| ターミナルが変な状態になった | ターミナルを強制リセットする | **`Ctrl + C`** を2回押す |

::: tip まずは `Esc`、ダメなら `Ctrl + C`
どちらを押しても壊れることはありません。安心して押してください。
:::

## インストール関連

### `npm: command not found` と出る

Node.js がインストールされていません。[ステップ2](/02-tools-install/) の **「Node.js を入れる」** の手順をやってください。

### Node.jsを入れたのに `npm: command not found` と出る

Node.jsをインストールした直後は、ターミナルがまだ認識していないことがあります。

**対処法：** ターミナルを一度閉じて、もう一度開き直してください。

### `permission denied` と出る（Mac）

[ステップ2](/02-tools-install/) の「Claude Code をインストールする」にある方法①または方法②を試してください。

### `xcode-select: note: No developer tools were found` と出る（Mac）

Gitが使えない状態です。[ステップ2](/02-tools-install/) の「Gitを入れる」の Mac の手順を実行してください。

## エラーが大量に出てパニックになった

::: danger エラーメッセージをターミナルにコピー＆ペーストしないでください！
エラーの文字列がそのまま「コマンド」として実行されてしまい、エラーがさらに増えて悪循環になります。
実際にこれでハマった事例があります。
:::

**対処法：**
1. まず **`Ctrl + C`** を何回か押してターミナルを落ち着かせる
2. エラー画面の **スクリーンショット** を撮る
3. Slackで相談してください

## Git関連

### `fatal: not a git repository` と出る

ターミナルのカレントディレクトリがプロジェクトフォルダではありません。

**対処法：** `cd` コマンドでプロジェクトフォルダに移動してください。

```bash
cd ~/Documents/プロジェクト名
```

### プッシュ時に認証エラーが出る（Mac）

**対処法：** ユーザー名とTokenを再入力してください。
- ユーザー名：GitHubのユーザー名
- パスワード欄：**Token**（`ghp_` で始まる文字列）を貼り付け

### コンフリクト（衝突）が起きた

2人が同じファイルの同じ場所を編集するとコンフリクトが起きます。

**対処法：** Claude Code に「コンフリクトを直して」と言えば解決してくれます。

## その他

### 何をしていいか分からなくなった

1. まず **`Ctrl + C`** でターミナルをリセット
2. Slackで状況を相談してください
